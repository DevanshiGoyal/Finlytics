## FinSight: Financial Predictive Analytics

End-to-end financial analytics project that combines supervised learning, forecasting, and anomaly detection in one interactive Streamlit dashboard. The system helps users estimate loan default risk, borrower churn risk, expected lending volume, and grade-level demand trends, while also supporting campaign optimization and unusual deposit detection workflows. It solves the problem of fragmented model demos by providing one integrated interface for risk scoring, monitoring, and scenario analysis. Intended users include data science learners, financial analytics practitioners, hackathon judges, and stakeholders who want business-facing ML outputs.

## Features

The features below are implemented in the current codebase and are available through the dashboard.

- Loan Default Risk module with interactive borrower inputs and probability-based risk output.
- Borrower Churn module to estimate likelihood of customer non-return after repayment.
- Loan Volume Forecast module with historical trend visualization and 3-month forecast table.
- Credit Demand by Grade module with model comparison, multi-grade trend plots, and heatmaps.
- Portfolio Intelligence Hub with:
	- Stress scenario testing (rate shock, income drop, DTI bump).
	- Batch portfolio scoring from uploaded CSV.
	- Explainability views (global and local feature impact).
	- Drift monitoring on selected numeric risk features.
	- Executive report export in Markdown format.
- Bank Term Deposit AI module with:
	- Tri-model training and leaderboard (Logistic Regression, Decision Tree, Random Forest).
	- Interactive prediction form.
	- Optional retraining on uploaded bank campaign CSV.
	- Feature importance charting.
- Deposit Anomaly module with:
	- Hybrid anomaly engine (Isolation Forest + MLP reconstruction error).
	- Live single-transaction risk scan.
	- Batch anomaly scoring and CSV export.
	- Optional retraining on uploaded anomaly dataset.

## Install And Run Instructions

### Prerequisites

- Python 3.10 to 3.12 recommended.
- Git installed.
- Windows, macOS, or Linux terminal.

### 1) Clone The Repository

```bash
git clone https://github.com/dalaldia5/Financial_Predictive_Analytics.git
cd Financial_Predictive_Analytics
```

### 2) Create And Activate Virtual Environment

Windows (PowerShell):

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
```

macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3) Install Dependencies

Standard install:

```bash
pip install -r requirements.txt
```

If you hit Windows long-path issues because of Jupyter packages, install runtime-only dependencies for dashboard execution:

```powershell
Get-Content requirements.txt | Where-Object { $_ -notmatch '^(jupyter|ipykernel)==' } | Set-Content requirements.runtime.txt
pip install -r requirements.runtime.txt
```

### 4) Optional Data Pipeline Run (Lending Club Raw Data)

Download the Lending Club accepted loans CSV and place it at:

- data/raw/accepted_2007_to_2018Q4.csv

Then run:

```bash
python src/data_pipeline.py
```

This generates processed data used by forecasting visuals when available.

### 5) Start The Dashboard

```bash
streamlit run dashboard/app.py
```

If port 8501 is already in use:

```bash
streamlit run dashboard/app.py --server.port 8502
```

## Usage Examples

### Example 1: Launch And Access The App

```bash
streamlit run dashboard/app.py
```

Expected outcome:

- Browser opens dashboard with 7 tabs:
	- Loan Default Risk
	- Borrower Churn
	- Loan Volume Forecast
	- Credit Demand by Grade
	- Portfolio Intelligence Hub
	- Bank Deposit AI
	- Deposit Anomaly

### Example 2: Score A Portfolio Batch

1. Open Portfolio Intelligence Hub.
2. Download sample portfolio CSV.
3. Re-upload it using Upload portfolio CSV.
4. Review computed fields such as:
	 - default_probability
	 - churn_probability
	 - risk_score
	 - risk_band
5. Export results with Download scored portfolio.

Sample output behavior:

- Metrics cards show average default/churn probability and high-risk share.
- Table shows scored records with risk band labels.

### Example 3: Anomaly Batch Detection

1. Open Deposit Anomaly tab.
2. Upload CSV with required columns:
	 - amount
	 - hour
	 - day_of_week
	 - frequency
3. Run batch scoring.
4. Download scored anomaly batch.

Sample output behavior:

- Flagged share displayed as percentage.
- Output includes anomaly_score, is_anomaly, reasons, and risk_label.

## Architecture Notes

- Frontend and orchestration are handled in a single Streamlit app.
- Domain logic is separated into reusable modules in src/.
- Pretrained models are loaded from local model artifacts in models/.
- Data is sourced from local CSV files; fallback synthetic/demo generation is used where appropriate.

## Limitations

- Some dashboard sections depend on locally available model and data artifacts.
- Full notebook support may fail on some Windows setups without long-path support.
- Forecast figures and model metrics are tied to the prepared datasets and assumptions in this repository.
- No production API or persistent data service is included.
